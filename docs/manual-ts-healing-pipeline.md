# Manual TS healing pipeline (AS3 → TS)

Цель: итеративно "долечить" автоконвертированные TS-файлы так, чтобы игра запускалась в браузере и повторяла поведение оригинального Flash-рантайма 1:1.

## Быстрый старт

```bash
npm run pipeline:heal
```

Скрипт выполнит:

1. конвертацию и автофиксы (`convert:full` + post-fix steps),
2. сбор статистики ошибок TypeScript,
3. перестроение runtime-манифеста Phaser,
4. smoke-проверку браузерной сборки (`npm run build`),
5. генерацию сводки в `tmp/manual-heal-summary.md`.

## Режимы запуска

По умолчанию обрабатывается весь проект (`all`). Можно ограничить область:

```bash
bash tools/as3-to-ts/manual-heal-pipeline.sh core
bash tools/as3-to-ts/manual-heal-pipeline.sh agi
```

Полезные env-переключатели:

- `SKIP_CONVERT=1` — не пересобирать TS заново (для быстрых ручных итераций),
- `BUILD_MANIFEST=0` — пропустить rebuild runtime-манифеста,
- `SUMMARY_FILE=tmp/custom-summary.md` — путь для итоговой сводки.

Пример:

```bash
SKIP_CONVERT=1 BUILD_MANIFEST=0 npm run pipeline:heal
```

## Ручной цикл исправлений (основной workflow)

После каждого запуска `pipeline:heal`:

1. Открой `tmp/tsc-manual-heal.stats.txt`.
2. Возьми верхнюю пару `файл + код ошибки`.
3. Найди оригинальный ActionScript-класс и сверяй логику по функции (не "массовыми" правками по всему файлу).
4. Исправь только выбранный узел (переменная, сигнатура, вызов, скобки, `this`, типы коллекций и т.д.).
5. Проверки:
   - `npm run typecheck`
   - `npm run build`
6. Повторяй, пока:
   - TypeScript не компилируется без критических ошибок,
   - браузерная сборка стабильно запускается,
   - ключевые игровые сценарии соответствуют Flash-оригиналу.

## Что считать "готовностью 1:1"

- Полный запуск без runtime-crash в браузере.
- Сохранена логика gameplay, математика, тайминги, переходы экранов.
- AGI-ветка и основной game-loop работают одинаково с оригиналом.
- Набор ключевых регрессионных сценариев проходит после каждой серии правок.
