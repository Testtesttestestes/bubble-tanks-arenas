# Manual TS healing pipeline — журнал исправлений

Дата: 2026-03-18 (UTC).

## Что запускалось

1. `npm run pipeline:heal` (полный прогон `manual-heal-pipeline.sh`, target=`all`).
2. Анализ проблем из `tmp/tsc-manual-heal.stats.txt` и ошибок в сгенерированных `migrated-ts/*.ts`.
3. Точечные правки в скрипте конвертации `tools/as3-to-ts/convert-as3-to-ts.js` + тесты.

## Успешные фиксы

### 1) Исправлен ложный heal `obj.0 -> obj[0]`, который ломал десятичные литералы

**Проблема:** правило для numeric property access заменяло `0.9` на `0[9]` и `0.1` на `0[1]`, что массово приводило к `TS1121`/`TS1489`.

**Фикс:** regex ограничен контекстом «после идентификатора/`]/`)`», чтобы лечить только `obj.0`, но не трогать обычные числа с плавающей точкой.

### 2) Исправлен regex для cast layout-присваиваний (`TS2540` heal)

**Проблема:** правило для `foo.alpha = ...` могло матчить середину цепочки и превращать
`this.var_80[param1].mc.alpha = ...` в
`this.var_80[param1].(mc as any).alpha = ...`.

**Фикс:** regex переписан с безопасным префиксом и поддержкой цепочек с индексами: теперь кастуется весь left-hand side (`(this.var_80[param1 as any].mc as any).alpha = ...`), без синтаксической порчи `.(...)`.

### 3) Добавлены регрессионные unit-тесты

Добавлены тесты, которые проверяют:
- что `0.9 + 0.1` не превращается в bracket-нотацию;
- что nested access `this.var_80[param1].mc.alpha = 0.5` не ломается в `.(mc as any)`.

## Неуспешные/частично успешные попытки

### 1) Полный `pipeline:heal` все еще не доводит проект до зеленого build

После прогона остались многочисленные ошибки TypeScript (в т.ч. синтаксические хвосты и AGI-ветка), а smoke-build Vite падает. Это ожидаемо для текущего объема ручного healing: исправления в этом коммите нацелены на **устранение ошибок самого пайплайна конвертации**, а не на полную ручную правку всего migrated-кода.
